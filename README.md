# yt-dlp Chrome Bridge

> [English version](README.en.md)

Связка из локального Python-сервера и расширения для Chrome, позволяющая
скачивать медиа через [yt-dlp](https://github.com/yt-dlp/yt-dlp) прямо
из браузера. Поддерживается **YouTube, Twitter/X, Twitch, SoundCloud,
Reddit, Instagram** для авто-определения URL текущей вкладки, плюс ещё
**1800+ сайтов** через ручную вставку ссылки или контекстное меню.

**Только для Windows.** Линукс/Мак не тестировались.

![License](https://img.shields.io/badge/license-MIT-blue.svg)

## Возможности

- **Попап расширения** с очередью, прогрессом и историей скачиваний.
- **Авто-определение** видео/плейлиста/канала на текущей вкладке для
  6 популярных сайтов.
- **Ручная вставка ссылки** для всех остальных сайтов из списка yt-dlp.
- **Контекстное меню**: правый клик по ссылке или странице → «Скачать
  через yt-dlp».
- **4 режима скачивания**: видео (с выбором качества), аудио (mp3/opus/m4a/wav),
  плейлист (с диапазоном), канал (последние N видео).
- **Гибкие паузы между загрузками**: из конфига yt-dlp / без пауз / свои
  N секунд.
- **Очередь** — одно скачивание за раз, чтобы не перегружать прокси.
  Поддержка отмены отдельной задачи или всей очереди разом.
- **Обновление yt-dlp в один клик** прямо из настроек расширения.
- **Прогресс в реальном времени**: имя файла, проценты, N/M в плейлистах.
- **Логи каждой задачи** в `logs/NNNN.log` с временными метками.
  Старые автоматически удаляются.
- **Глобальные настройки по умолчанию** (контейнер, паузы, формат) и
  per-task переопределение в попапе.

## Требования

- **Windows 10/11**
- **Python 3.8+** (стандартная библиотека, никаких pip-зависимостей)
- **yt-dlp.exe** — [скачать](https://github.com/yt-dlp/yt-dlp/releases/latest)
- **ffmpeg.exe + ffprobe.exe** — [скачать](https://www.gyan.dev/ffmpeg/builds/)
  (нужен для склейки видео+аудио и конвертации). Положи их рядом с
  `yt-dlp.exe` либо добавь папку с ffmpeg в системный PATH — yt-dlp найдёт
  оба варианта.
- **deno.exe** — [скачать](https://github.com/denoland/deno/releases/latest)
  (архив `deno-x86_64-pc-windows-msvc.zip`, **не** `denort`). Начиная с
  yt-dlp 2025.11.12 это **обязательная** зависимость для скачивания с
  YouTube — без deno YouTube либо отдаёт ущербные форматы, либо вообще
  не качается. Положи `deno.exe` рядом с `yt-dlp.exe` или добавь в PATH.
  Для других сайтов (Twitter/X, Twitch, SoundCloud и т.д.) deno не нужен.
- **Google Chrome / Chromium** (Edge тоже подойдёт)

## Установка

### 1. Подготовка папки yt-dlp

Создай где-нибудь папку (например `D:\yt-dlp`), положи туда:

- `yt-dlp.exe`
- `ffmpeg.exe`, `ffprobe.exe`
- `deno.exe` (нужен для YouTube начиная с yt-dlp 2025.11.12)
- `yt-dlp.conf` — твой главный конфиг yt-dlp (прокси, куки, шаблоны имён, паузы и т.д.)

Минимальный пример `yt-dlp.conf`:

```
-P "videos"
-P "audio:music"
--embed-metadata
--no-mtime
--sleep-interval 10
--max-sleep-interval 20
```

Расширение добавит свои флаги поверх (качество, формат, контейнер) — конфиг
менять не нужно.

#### Расширенный пример `yt-dlp.conf`

Этот конфиг я использую сам. Работает с Coala Clash на дефолтном порту 7897
(SOCKS5 системный прокси), использует куки Firefox для авторизации YouTube
аккаунта (нужно для скачивания возрастных видео и приватных плейлистов).

```
# coding: utf-8

# --- СЕТЬ И АВТОРИЗАЦИЯ ---
--proxy "socks5://127.0.0.1:7897"
--cookies-from-browser firefox
--socket-timeout 60

# --- БЕЗОПАСНОСТЬ (Для каналов и плейлистов) ---
# Снижаем скорость до 10МБ/с, чтобы не перегружать прокси
--limit-rate 10M
# Рандомная пауза между видео от 20 до 60 секунд (имитация человека)
--sleep-interval 20
--max-sleep-interval 60

# --- ПУТИ (относительные) ---
-P "./videos"
-o "audio:./music/%(title)s.%(ext)s"
-o "%(title)s.%(ext)s"

# --- АРХИВАЦИЯ ---
# Не качать видео, если его ID уже есть в этом файле
--download-archive "archive.txt"

# --- КАЧЕСТВО ---
-S "res,fps"
--merge-output-format mkv

# --- ЛОГИКА ДВОЙНОГО ФАЙЛА (MKV + MP4) ---
--exec "after_move:if '%(ext)s'=='mkv' ffmpeg -y -i %(filepath)q -c copy \"%(filepath.0:-4)s.mp4\""

# --- ДОПОЛНИТЕЛЬНО ---
--embed-subs
--sub-langs "all,-live_chat"
--embed-metadata
--ignore-errors
```

> **Почему именно Firefox?** Chrome и Edge с недавних пор шифруют куки
> (Application-Bound Encryption), и yt-dlp не всегда может их прочитать.
> Firefox — самый надёжный вариант. Если у тебя только Chrome, есть два
> пути: либо установить Firefox только для одной авторизации в YouTube
> (cookies-from-browser firefox после этого работает даже если Firefox
> закрыт), либо вручную экспортировать куки в файл и использовать
> `--cookies cookies.txt` вместо `--cookies-from-browser`.

### 2. Клонирование репозитория

```bash
git clone https://github.com/Nureble/yt-dlp-chrome-bridge.git
cd yt-dlp-chrome-bridge
```

### 3. Настройка сервера

Скопируй пример конфига и отредактируй под себя:

```bash
copy config.example.json config.json
```

Открой `config.json` в любом редакторе и укажи путь к папке с `yt-dlp.exe`:

```json
{
  "yt_dlp_dir": "D:/yt-dlp",
  "host": "127.0.0.1",
  "port": 5000,
  "keep_finished": 30,
  "log_retention_days": 30
}
```

> **Важно:** в `yt_dlp_dir` используй прямые слеши (`D:/yt-dlp`) либо
> двойные обратные (`D:\\yt-dlp`). Одинарные `\` JSON не поймёт.

### 4. Запуск сервера

Двойной клик по `start-server.bat`. В консоли должно появиться:

```
============================================================
 yt-dlp local bridge v2.6 (queue + settings + progress + update + sleep + logs)
============================================================
 yt-dlp.exe : D:/yt-dlp/yt-dlp.exe
 logs       : D:/yt-dlp/logs
 settings   : D:/yt-dlp/yt-dlp-bridge-settings.json
 Слушаю    : http://127.0.0.1:5000
```

Окно сервера должно оставаться открытым во время использования. Закроешь — расширение перестанет работать (но текущее скачивание не прервётся).

### 5. Установка расширения в Chrome

1. Открой `chrome://extensions/`
2. Включи **«Режим разработчика»** (правый верхний угол)
3. Нажми **«Загрузить распакованное»** и выбери папку `extension/` из репозитория

В панели Chrome появится иконка расширения. Если хочешь закрепить — клик по
паззлу справа от адресной строки → булавка рядом с «yt-dlp Bridge».

### 6. Проверка

Зайди на YouTube, открой любое видео, кликни иконку расширения. Должен
появиться попап с зелёной галочкой («Сервер работает») и подхваченным URL
видео. Жми «Скачать» — задача появится в очереди.

## Использование

### Попап на YouTube/Twitter/Twitch/SoundCloud/Reddit/Instagram

1. Открой страницу с видео/плейлистом/каналом
2. Клик по иконке расширения → URL и тип контента подхватятся автоматически
3. Выбери режим (видео/аудио/плейлист/канал) и параметры
4. «Скачать»

### Любой другой сайт

1. Скопируй ссылку на медиа
2. Клик по иконке расширения
3. Вставь URL в поле, выбери режим
4. «Скачать»

Полный список поддерживаемых сайтов: [supportedsites.md](https://github.com/yt-dlp/yt-dlp/blob/master/supportedsites.md).

### Правый клик

На странице любого поддерживаемого сайта — правый клик → «Скачать эту
страницу через yt-dlp». На ссылке — «Скачать через yt-dlp». Скачивание с
дефолтными настройками без открытия попапа.

### Глобальные настройки

Иконка расширения → ⚙ (или `chrome://extensions/` → «Параметры расширения»).
Можно настроить:

- Качество видео по умолчанию
- Аудио-формат и битрейт по умолчанию
- Контейнер: MKV+MP4 / только MP4 / только MKV
- Использовать ли `archive.txt` по умолчанию
- Паузы между загрузками
- **Обновить yt-dlp** одним кликом

## Куда сохраняются файлы

Зависит от твоего `yt-dlp.conf`. По дефолту с минимальным конфигом выше:

- Видео: `<yt_dlp_dir>/videos/`
- Аудио: `<yt_dlp_dir>/music/`
- Плейлист: `<yt_dlp_dir>/videos/playlist/<название>/01 - трек.mp3`
- Канал: `<yt_dlp_dir>/videos/channel/<имя_канала>/название.mkv`

> **Важно про `-o` в твоём `yt-dlp.conf`.** Для режимов «Плейлист» и
> «Канал» расширение добавляет свой шаблон `-o` который **переопределяет**
> шаблон из конфига. Это нужно для создания подпапки с названием
> плейлиста/канала и для нумерации треков. Шаблон `-o` из твоего конфига
> при этом продолжает работать для одиночных видео и аудио.

## Решение проблем

### При установке

**`python` не является внутренней или внешней командой.** При установке
Python ты не отметил галочку «Add python.exe to PATH». Скачай Python
заново с [python.org](https://www.python.org/downloads/), на первом
экране установщика поставь галку **«Add python.exe to PATH»** и переустанови.

**Сервер сразу падает с `[!] FATAL: yt_dlp_dir = ... — папка не существует`.**
Путь из `config.json` указывает на несуществующую папку. Создай её или
поправь путь в `config.json` (помни про прямые слеши `D:/yt-dlp` либо
двойные обратные `D:\\yt-dlp`).

**`[!] FATAL: ни config.json, ни config.example.json не найдены`.** Ты
запускаешь `start-server.bat` не из папки репозитория. Двойной клик по bat
работает только если он в корне рядом с `server.py`.

**При первом запуске `[!] ОТКРОЙ config.json И УКАЖИ СВОЙ yt_dlp_dir`.**
Это нормальное сообщение при первом старте — сервер скопировал пример
конфига. Открой `config.json`, поправь путь, перезапусти `start-server.bat`.

### При работе

**В попапе «Сервер не отвечает».** Запусти `start-server.bat`. Если запущен —
проверь что в консоли сервера нет ошибок и что порт 5000 не занят.

**Расширение работает, но при скачивании ошибка `yt-dlp.exe не найден`.**
Проверь что в `config.json` правильный путь и что в этой папке реально
лежит `yt-dlp.exe`.

**Кнопки расширения не работают на странице YouTube.** Перезагрузи вкладку
(F5). Если не помогло — `chrome://extensions/` → ↻ у расширения.

**В именах файлов появляются `?????` вместо кириллицы.** Это лечится
обновлением yt-dlp (настройки расширения → «Обновить»). Сервер уже передаёт
`--encoding utf-8`, но старые версии yt-dlp могли не принимать его правильно.

**Очередь висит, не идёт дальше.** ffmpeg застрял на интерактивном вопросе.
Открой Диспетчер задач, прибей `ffmpeg.exe` и `yt-dlp.exe`. В свой
`yt-dlp.conf` добавь `-y` ко всем `--exec` где запускается ffmpeg, чтобы он
молча перезаписывал существующие файлы.

**Куда смотреть логи задачи.** В `<yt_dlp_dir>/logs/NNNN.log` — лог именно
этой задачи с временными метками и полным выводом yt-dlp.

**В логе предупреждение `No supported JavaScript runtime could be found`
при скачивании с YouTube.** В папке `yt_dlp_dir` нет `deno.exe` или его
версия слишком старая. Скачай свежий с
[github.com/denoland/deno/releases](https://github.com/denoland/deno/releases/latest)
(архив `deno-x86_64-pc-windows-msvc.zip`, не `denort`!), распакуй
`deno.exe` в свою папку yt-dlp.

**В логе `Some web client https formats have been skipped` или
`YouTube is forcing SABR streaming for this client`.** Тот же симптом —
yt-dlp не может прорешать JS-челлендж YouTube. Нужен deno (см. выше) или
обновление yt-dlp (через настройки расширения → «Обновить»).

**Скорость скачивания YouTube стала очень низкой (50–100 KiB/s).** YouTube
душит ботов. Без deno и/или с устаревшим yt-dlp скорость падает до плинтуса.
Решение — установить deno и обновить yt-dlp.
